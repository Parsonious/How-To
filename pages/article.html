How To: Add Roles To A User In EMIT

Brief Description of This Task

Users in EMIT require Technical Support intervention in order to access more than the basic Examiner Queue. This task involves running a SQL Select statement and a series of Insert statements in order to provide the specified EMIT User(s) with the requested access level. While originally development was intended to be made to allow ESIS to perform this task themselves, EMIT can be buggy at times. This can be due in part to its older framework so it remains likely that this task will continue to be a required Technical Support task for a while yet.

Task Prerequisites

  • Read and Write privileges on the GalantUT.dbo.ESIS database.
  • The ticket Opener must provide the Examiner Code for any Examiner that needs to be updated

Identify The Ticket

Identifying these tickets can be quite simple. The title of the ticket typically falls within the range of the titles found in the linked sample tickets. For instance: "EMIT - User Needs Access To Operations Queues" as seen in ticket number 99463 indicates that the EMIT user does not currently have access to the mentioned queue and needs access granted by a member of the Technical Services team.

Writing Your Sql

First you want to identify what roles already exist under the given user. This can be accomplished with the following SQL script:

USE ESIS
SELECT * FROM 
portal.ExaminerRole ER join portal.Role R
	on ER.RoleID = R.RoleID
Join portal.Examiner E
	on E.ExaminerID = ER.ExaminerID
Join portal.Login L
	on E.LoginID = L.LoginID
Where E.ExaminerCode = '' --take out hyphen
order by L.FirstName

Once you've identified the roles that are present, you will need to take note of the ExaminerID column that is present in this original Select statement. That is the ID used for the rest of the task.

Roles Needed For Individual Queues

    Internal Queue
  • 27: Examiner
  • 28: Manager
  • 29: ESISClaims
  • 39: AdminLimited
    Operations Queue
  • 30: Unassigned
  • 31: Research
  • 32: NoMatch
  • 37: RecordOnly

After identifing what is needed to give the requested user access you must write your SQL insert statements to make the changes.

Insert portal.ExaminerRole
Select '6614','32' 
Insert portal.ExaminerRole
Select '6614','30'
Insert portal.ExaminerRole
Select '6614','31'
Insert portal.ExaminerRole
Select '6614','37'

In the above example the Examiner ID 6614 is being given access to the Operations Queue.

Key Notes

  • Role 27 always exists and should not be removed
  • Duplicate Roles Should NOT be added

Update Your Ticket

As always the final step to any ticket is to update the ticket opener and note the SQL you used!
